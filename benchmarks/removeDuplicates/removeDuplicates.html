Remove Duplicates (RDUPS):

Given a sequence of elements of any (uniform) type, a hash function on the type, and an equality operation, remove any duplicates. The hash is defined to return an unsigned integer and will always return the same integer for equal elements. The benchmark should allow auxiliary data simply by defining equality on just the key.

The code must not take advantage of the specific type beyond the equality and hash functions. If the size is used the benchmark should be reported with -Z.

Input and Output File Formats

The input and output should be in the sequence format. The elements must be returned so that the first equal element in the input sequence is maintained and the others are removed (unless indicated with -X). Note that this only makes a difference if there is auxiliary data.

Default Input Distributions

Each distribution should be run for n=10,000,000. The weights used for average time are given in parentheses.
  • (1) 32-bit unsigned integer keys from the uniform distribution.
    randomSeq -t int <n> <filename>
  • (1) Integer keys randomly and uniformly selected from the range [0..100000).
    randomSeq -t int -r 100000 <n> <filename>
  • (1) 32-bit unsigned integer keys from the exponential distribution.
    exptSeq -t int <n> <filename>
  • (3) strings selected from the trigram distribution.
    trigramSeq <n> <filename>
  • (3) strings selected from the trigram distribution tagged with auxiliary 32-bit integers.
    trigramSeq <n> <tmpname>
    addDataSeq -t int <tmpname> <filename>
